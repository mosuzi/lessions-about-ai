# INT4、INT8、FP8、FP16、BF16、FP32 都是啥

更新时间：2026-06-18

这几个词说的是数值在计算机里用多少 bit、用什么格式来表示。它们不是“模型类型”，而是模型权重、激活值、梯度、缓存等张量数据的存储和计算精度。

简单说：

- bit 越少，显存/内存越省，带宽压力越小，通常推理更快。
- bit 越少，表示范围或精度越弱，模型质量、训练稳定性、数值溢出风险更敏感。
- 整数格式更适合压缩权重和推理；浮点格式更适合保留动态范围，训练更常用。

## 基础概念

### 整数和浮点

INT 是 integer，整数。INT8 表示一个数用 8 bit 表示，INT4 表示一个数用 4 bit 表示。整数格式本身没有小数点，LLM 里常通过量化参数把真实小数映射到整数，例如：

```text
real_value ~= scale * (int_value - zero_point)
```

FP 是 floating point，浮点数。浮点数把一个数拆成符号、指数、尾数，类似科学计数法：

```text
value ~= sign * significand * 2^exponent
```

所以浮点数可以表示很大或很小的数，但同样 bit 数下精度未必比整数高。

### 权重、激活、梯度、KV cache

在大模型里，“用什么精度”可能指不同对象：

- 权重：模型参数。量化权重最常见，因为它直接决定模型文件大小和显存占用。
- 激活值：模型中间层输出。激活量化更难，因为它随输入变化。
- 梯度：训练时的反向传播信号。训练通常比推理更依赖 FP16/BF16/FP8 这类浮点格式。
- KV cache：自回归生成时保存 attention 的 key/value。长上下文推理时 KV cache 可能成为显存大头。

一个模型被称为“INT4 模型”，很多时候只是“权重量化到 INT4”，不代表所有计算都用 4 bit。

## FP32

FP32 是 32 bit 浮点数，也叫 single precision，IEEE 754 里对应 binary32。典型结构是：

| 部分 | bit 数 |
| --- | ---: |
| 符号位 | 1 |
| 指数 | 8 |
| 尾数/有效数存储位 | 23 |

实际有效精度约 7 位十进制数字。

特点：

- 数值范围大，稳定性好。
- 训练早期神经网络时的默认精度。
- 显存和带宽成本高，现在大模型训练/推理一般不会全程 FP32。
- 常用于优化器状态、关键累加、归一化、loss 计算等对稳定性敏感的位置。

适合：

- 需要高稳定性的数值计算。
- 调试基准。
- 小模型或成本不敏感的训练。

不适合：

- 大模型全量训练和推理的主格式，因为太占显存和带宽。

## FP16

FP16 是 16 bit 浮点数，也叫 half precision，IEEE 754 里对应 binary16。典型结构是：

| 部分 | bit 数 |
| --- | ---: |
| 符号位 | 1 |
| 指数 | 5 |
| 尾数/有效数存储位 | 10 |

实际有效精度约 3 到 4 位十进制数字。

特点：

- 比 FP32 省一半显存，吞吐更高。
- 曾是深度学习混合精度训练的主力格式。
- 指数位少，动态范围比 FP32 小，训练时容易遇到 underflow/overflow。
- 常配合 loss scaling 使用，避免梯度太小被舍入成 0。

适合：

- GPU 支持良好的训练和推理。
- 对速度和显存敏感，但还需要浮点动态范围的场景。

不适合：

- 数值范围特别敏感的训练链路。
- 不做混合精度保护的直接全 FP16 训练。

## BF16

BF16 是 16 bit 浮点数，全名是 bfloat16。它和 FP16 一样只用 16 bit 存一个数，但把更多 bit 留给指数范围：

| 部分 | bit 数 |
| --- | ---: |
| 符号位 | 1 |
| 指数 | 8 |
| 尾数/有效数存储位 | 7 |

它的指数位和 FP32 一样多，所以动态范围接近 FP32；尾数位比 FP16 少，所以精度比 FP16 更粗。

特点：

- 动态范围大，训练时比 FP16 更不容易 underflow/overflow。
- 通常不需要像 FP16 那样强依赖 loss scaling。
- 精度低于 FP16，但大模型训练往往更看重稳定的动态范围。
- 需要硬件和框架支持，现代数据中心 GPU/TPU 上更常见。

适合：

- 大模型预训练和全量微调。
- 需要混合精度加速，但又希望减少 FP16 数值稳定性问题的训练链路。
- 支持 BF16 的 GPU/TPU 环境。

不适合：

- 不支持 BF16 加速的老硬件。
- 对尾数精度非常敏感的小规模数值计算。

## FP8

FP8 是 8 bit 浮点数。它不是传统 IEEE 754 基础格式，AI 训练/推理里常见的 FP8 有两种编码：

| 格式 | 指数 | 尾数 | 大致取向 |
| --- | ---: | ---: | --- |
| E4M3 | 4 bit | 3 bit | 精度稍好，范围较小 |
| E5M2 | 5 bit | 2 bit | 范围更大，精度更低 |

FP8 的目标是进一步降低显存和带宽，同时保留浮点数的动态范围优势。相比 INT8，FP8 在某些训练或激活值场景更自然；相比 FP16/BF16，它更省资源，但数值更脆弱。

特点：

- 显存占用约为 FP16 的一半。
- 在支持 FP8 Tensor Core/矩阵引擎的硬件上吞吐很高。
- 常需要 per-tensor 或 per-channel scaling，避免数值超出可表示范围。
- 更依赖框架、硬件和训练 recipe，不能简单把 FP16 替换成 FP8 就完事。

适合：

- 新硬件上的大模型训练加速。
- 经过校准的推理。
- 对吞吐和成本极敏感的部署。

不适合：

- 缺少 FP8 硬件支持的环境。
- 没有校准/缩放策略的朴素训练。
- 对极小质量波动非常敏感但没有充分 eval 的上线链路。

## INT8

INT8 是 8 bit 整数量化。常见取值范围：

- signed INT8：-128 到 127。
- unsigned UINT8：0 到 255。

在 LLM 里，INT8 常用于权重量化，有时也用于激活量化。核心不是“模型真的只会整数”，而是用 scale/zero point 把连续小数近似成整数，再在计算时配合反量化或整数矩阵乘法。

特点：

- 模型显存占用约为 FP16 的一半。
- 一般质量损失较小，工程成熟度高。
- 对离群值敏感，尤其是激活值里出现极端大值时。
- 权重 INT8 通常比权重+激活全 INT8 更容易保持质量。

适合：

- 推理部署。
- 想在质量损失较小的情况下省显存。
- CPU、GPU、NPU 等多种硬件上的通用量化。

不适合：

- 对数值分布变化很大的训练主流程。
- 没有校准集、没有 eval 的直接生产替换。

## INT4

INT4 是 4 bit 整数量化。4 bit 能表示的状态非常少：

- signed 4 bit 常见范围：-8 到 7。
- unsigned 4 bit 常见范围：0 到 15。

LLM 本地部署里常见的 4-bit quantization 多数是权重量化，例如 GPTQ、AWQ、GGUF/llama.cpp 的 Q4 系列、bitsandbytes NF4 等。它大幅降低模型大小，让消费级显卡甚至纯 CPU 跑更大的模型成为可能。

特点：

- 显存占用约为 FP16 的四分之一。
- 可以让更大参数量模型塞进同一块显卡。
- 质量损失比 INT8 更明显，尤其是小模型、复杂推理、代码、数学、长上下文任务。
- 更依赖量化方法、group size、校准数据和具体模型结构。

适合：

- 本地推理。
- 显存非常有限但希望运行较大模型。
- 交互式聊天、草稿、轻量 agent 等可容忍少量质量损失的任务。

不适合：

- 高可靠生产任务但没有充分 eval。
- 需要严格数值一致性的场景。
- 大规模训练主流程。

## 横向对比

| 格式 | 类型 | bit | 主要用途 | 优点 | 风险 |
| --- | --- | ---: | --- | --- | --- |
| FP32 | 浮点 | 32 | 稳定训练、累加、基准 | 稳定、范围大、精度高 | 慢、占显存 |
| FP16 | 浮点 | 16 | 混合精度训练/推理 | 快、省显存 | 动态范围较小 |
| BF16 | 浮点 | 16 | 大模型训练/微调 | 动态范围大、训练更稳 | 尾数精度较低、依赖硬件 |
| FP8 | 浮点 | 8 | 新硬件训练/推理加速 | 更快、更省带宽 | 依赖硬件和缩放策略 |
| INT8 | 整数 | 8 | 推理量化 | 成熟、质量损失小 | 离群值和激活量化难 |
| INT4 | 整数 | 4 | 极限压缩、本地推理 | 显存省很多 | 质量更容易下降 |

## 选择建议

如果你是在用模型：

- 云端 API：一般不用关心这些，供应商已经处理好了。
- 本地跑模型：优先试 INT4 或 INT8，看显存和质量是否满足。
- 质量优先：选 FP16/BF16 或高质量 INT8。
- 显存优先：选 INT4，但要接受质量波动。
- 做严肃 benchmark：同一任务、同一提示词、同一采样参数下比较，不要只看模型能不能启动。

如果你是在训练或微调：

- 全量训练：常见是 BF16/FP16 混合精度，FP8 取决于硬件和框架。
- LoRA/QLoRA：底座可以 4-bit 量化，训练小 adapter。
- 不要只看“bit 数”：训练稳定性往往由优化器、归一化、loss scaling、梯度累加、数据质量一起决定。

## 常见误解

### 误解一：INT4 一定比 INT8 差很多

不一定。好的 4-bit 方法、合适的模型和任务，可能质量损失很小。但一般来说，bit 越低，风险越高，需要 eval。

### 误解二：FP8 就是 INT8 的浮点版，随便替换即可

不是。FP8 需要缩放、校准和硬件支持。训练时尤其需要 recipe，不是改个 dtype 就能稳定。

### 误解三：量化只影响模型大小

还会影响速度、显存带宽、KV cache、算子支持、困惑度、指令遵循、长上下文稳定性和工具调用准确率。

### 误解四：模型卡住显存就只能换小模型

不一定。可以考虑 INT4/INT8 权重量化、KV cache 量化、降低上下文长度、降低 batch size、使用 CPU offload 或换推理框架。
